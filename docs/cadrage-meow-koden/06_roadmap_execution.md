# 06 - Roadmap d'exécution

## Phase 0 - Cadrage consolidé

**Objectif** : aligner vision, scope, modèle d'offre, besoins du premier client et principes de généralisation.

**Livrables** : pack de cadrage, registre des décisions, cartographie des besoins client, premiers critères de passage.

**Critères de passage** : priorités validées, palier 1 confirmé, responsabilités Koden clarifiées.

## Phase 1 - Socle client industrialisable

**Objectif** : stabiliser Meow pour le premier client et livrer les axes prioritaires.

**Livrables** : configuration des données commerciales, carnet d'adresses cadré, reporting/pilotage utile, exports robustes, hébergement opéré.

**Critères de passage** : usage réel possible par le client, données exportables, incidents bloquants traités, support initial en place.

## Phase 2 - Offre réutilisable

**Objectif** : transformer le retour du premier client en offre répétable.

**Livrables** : processus d'onboarding, modèle de configuration standard, documentation d'exploitation, support structuré, proposition commerciale cadrage/adaptation + abonnement.

**Critères de passage** : deuxième client adressable sans refonte, coût d'adaptation maîtrisé, périmètre de support explicite.

## Phase 3 - Plateforme opérée

**Objectif** : préparer une exploitation plus mutualisée et scalable.

**Livrables** : architecture cible de mutualisation, supervision renforcée, séparation des environnements, gouvernance produit/run, plan d'intégrations.

**Critères de passage** : mutualisation techniquement cadrée, risques sécurité maîtrisés, modèle économique récurrent confirmé.
