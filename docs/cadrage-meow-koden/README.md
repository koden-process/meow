# Pack de cadrage Meow / Koden

## Objet du pack

Ce pack cadre la poursuite de Meow par Koden comme une initiative produit-service. Il sert de base commune pour aligner vision, métier, produit, technique, gouvernance, trajectoire de delivery et run.

## Positionnement retenu

Meow est une solution de pilotage commercial pour fabricants, distributeurs et fournisseurs d'équipements B2B, opérée par Koden, permettant de structurer les opportunités, les comptes, les étapes de vente, les prévisions et les échanges entre équipes dans un outil configurable, industrialisable et adapté aux cycles commerciaux réels.

## Palier 1 retenu

**Palier 1 - Socle client industrialisable** : stabiliser Meow autour de la configuration des données commerciales, du pilotage du pipeline et de l'export/intégration, afin de répondre au premier client tout en préparant une offre réutilisable pour d'autres organisations industrielles B2B.

## Ordre de lecture recommandé

1. `00_executive_summary.md` pour comprendre la logique stratégique.
2. `02_vision_projet.md` et `03_cadrage_metier.md` pour comprendre l'ambition et les besoins métier.
3. `04_modele_cible.md` pour stabiliser les objets de référence.
4. `05_trajectoire_produit_technique.md`, `06_roadmap_execution.md` et `11_scope_et_non_scope.md` pour comprendre la trajectoire.
5. `07_decisions_structurantes.md`, `08_registre_adr_metier.md`, `09_risques_et_hypotheses.md` et `10_indicateurs_de_succes.md` pour piloter les arbitrages.
6. `12_plan_de_lancement.md`, `13_gouvernance_et_raci_detaille.md` et `14_cadrage_technique.md` pour organiser le lancement, le build et le run.

## Usage recommandé

Ce pack doit être mis à jour par itérations. Les décisions doivent rester explicites, datées si nécessaire, et séparées des points ouverts. Les spécifications fonctionnelles détaillées, tickets de développement et ADR techniques doivent être produits ensuite à partir de ce cadrage, sans surcharger ces documents.
